**프로젝트 설명**  

 ‘**기후 변화**’라는 큰 주제를 분석 시작

 →‘**산림 파괴**’, ‘**탄소 배출**’, ‘**쓰레기 배출**’ 등의 작은 키워드 추출

 → 가장 생활에 밀접하다고 생각한 ‘**쓰레기 배출**’ 선택

 > 쓰레기 배출 문제의 핵심은 무엇인가?

 →쓰레기 문제의 해결 방법인 **재활용**이 되지 않고 있음

 > 쓰레기 재활용률이 낮은 이유는 무엇인가?

 → 잘못된 방법의 쓰레기 분리 배출

 ->**잘못된 쓰레기 분리 배출** 을 문제로 선택

학교 SW/AI 문제해결 프로젝트로 시작된 제로 웨이스트 주제 활동입니다.  
‘기후 변화’라는 큰 주제에서 **‘잘못된 쓰레기 분리 배출’** 문제를 추출하고,  
이를 **AI 이미지 인식**을 통해 해결하는 방식을 선택했습니다.

- 재활용이 잘 안 되는 이유: 분리 배출이 잘못되었기 때문  
- 이를 해결하기 위해 **AI가 쓰레기 종류를 인식**하고,  
  **각 쓰레기에 맞는 분리배출 방법을 알려주는 앱**을 개발했습니다.

**주요 기능**:  
- 버튼 클릭 시 종류별 분리배출 방법 안내  
- AI가 카메라를 통해 쓰레기 종류를 자동 인식하고 분리법 제안  
- 직관적인 UI로 사용자 접근성 향상  
- 분리수거에 익숙하지 않은 학생 사용자도 쉽게 사용 가능

### 2. 학교용 분실물 찾기 앱 개발

- **기간**: 2025.03.24 ~ 2025.04.11  
- **맡은 역할**: 전체 기획 및 디자인, 프론트엔드와 백엔드 포함한 전체 개발  
- **사용 기술**:
  - **호스팅**: Firebase Hosting  
  - **프론트엔드**: HTML, CSS (Vive), JavaScript, Material Design 기반 UI, AI (Vive)  
  - **백엔드**: Firebase 실시간 데이터베이스 또는 Firestore, JavaScript, Vive  

---

**프로젝트 설명**  
이 프로젝트는 **제46회 서울특별시 강남서초학생과학발명품경진대회**에 출품한  
**디지털 분실물 관리 시스템**으로,  
학생들이 학교에서 분실한 물품을 쉽고 효율적으로 찾을 수 있도록 설계된 **웹 기반 앱**입니다.

- 사용자: 1층~4층 중 하나를 선택하여 분실물 등록  
- 입력 항목: 이미지와 이름 → 시각적 정보 제공  
- 기능: 간단한 삭제 기능 → 관리 편의성 향상  
- 실시간 동기화: Firebase를 통해 여러 기기와 즉시 정보 공유  
- UI/UX: 접근성 향상을 위해 버튼 크기와 레이아웃 최적화  

이 앱은 실체 없는 소프트웨어 형태로 구현되어  
**기존 아날로그 시스템에서 디지털로의 전환**이라는 실용성과,  
**소프트웨어 기반 발명품**으로서의 독창성을 함께 갖추고 있습니다.

또한 각 층마다 분실물 관리소가 있다는 가정 하에,  
**교무실 방문 없이 검색창으로 분실물 위치를 확인**할 수 있게 하여  
탐색 시간 절감과 만족도 향상을 기대할 수 있습니다.

> 처음으로 **Vive 기술**을 도구로 사용해보며 UI 구성에 활용했습니다.

궁극적으로 이 시스템은 **도서관, 공공기관 등 다양한 장소**에도  
구조만 수정하여 적용 가능한 **보편적인 디지털 관리 솔루션**이 될 수 있습니다.

---

**소감 / 배운 점**  
- **Vive**를 사용하니 빠르게 디자인을 구현할 수 있었습니다.  
- 그러나 **오류 확인이 어렵고 창의성 저하** 가능성 존재하여 AI는 아직은 보조 도구로만 활용하는 것이 좋겠다고 생각하게 되었습니다.  
- **Firebase는 사용하기 그나마 쉬웠지만 보안쪽을 공부하지 못하고 사용하다 보니 보안상 주의 필요함을 알 수 있게 되었습니다.** (예: API 키 노출) → 차후 다른 배포 시스템 고려  
- **개발자는 도구를 활용하되 주체적으로 판단하고 개선하는 자세가 중요함**

### 3. Undertale 스타일 턴제 전투 게임

- **기간**: 2025.05.9 ~ 2025.05.10  
- **맡은 역할**: 전체 기획 및 디자인, 게임 로직 및 UI 포함한 전체를 AI로 개발  
- **사용 기술**:  
  - **프론트엔드**: HTML, CSS (Vive), JavaScript, Canvas API  
  - **게임 구조**: VIVE 코딩 환경 + 직접 만든 스프라이트(piskel)  
  - **이미지 에셋**: 직접 편집한 픽셀 아트 몬스터 이미지  

---

**📌 프로젝트 설명**  
이 프로젝트는 **턴제 전투 게임**을 Canvas API와 순수 JavaScript로 직접 구현한 것입니다.  
게임 구조는 평소 즐겨하던 게임인 “Undertale”에서 영감을 받아서 만든 간단한 턴제 전투 게임입니다.

- **Z 키**: 공격 & 시작(처음 시작시 z를 눌러야함) → 타이밍에 따라 데미지 결정(중간에 더 가까울때 z를 누르면 데미지가 더 들어감 )  
- **X 키**: 아이템 사용 (HP 회복)  
- **C 키**: 체력 일부와 맞바꾸는 강한 스킬 
- **스페이스바**: 게임 오버 시 재시작 버튼  
- **방향키 / WASD**: 회피및 이동

- 각 적은 고유한 **탄막 패턴**을 가짐
  - 고블린: 오른쪽 상단에서 퍼지는 느린 대각선 탄환  
  - 늑대: 상하 번갈아 집중 탄환 발사 (너무 쉬워서 밸런스 패치 고민중) 
  - 마법사: 회오리 형태 탄환  
  - 불: 아래에서 위로 솟아오르는 화염 탄  
  - 로봇: 큰 네모형 공격이 상하좌우로 순차적 발사 (피하기 어려워서 밸런스 패치 고민중)  
  - 슬라임: 매우 약하며 한 번의 공격으로 처치 가능

> 턴마다 적 패턴이 자동 실행되며, 피해를 회피하지 못하면 체력이 감소합니다.  
> 적을 처치할 경우 플레이어 체력이 **자동으로 최대치 회복**되어 다음 턴을 준비할 수 있습니다.

---

**소감 / 배운 점**  
- 처음으로 **Canvas API**와 JavaScript를 활용해 게임을 직접 설계하고,  
  애니메이션, 충돌 판정, 키 이벤트, 난이도 조정까지 **모든 전투 시스템을 직접 구현**해본 경험이었습니다.  
- **AI 도구(Vive)**를 활용하면 빠르게 구조를 만들 수 있지만,  
  **직접 디버깅하고 테스트하면서 논리와 코드를 정제하는 것이 가장 중요**하다는 점을 배웠습니다.  
- 실제로 고블린의 탄막 퍼짐, 슬라임 처치 후 체력 회복 같은 **디테일을 조정하는 경험**이 특히 재미있고 유익했습니다.
